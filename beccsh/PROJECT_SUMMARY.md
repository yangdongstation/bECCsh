## 🔧 技术特性

### 已实现的"功能"
1. **六层熵收集系统**
   - 键盘输入熵（带时序）
   - CPU抖动熵
   - 进程树熵
   - 系统状态熵
   - 网络统计熵
   - 哈希链混合

2. **完整ECC实现**
   - secp256r1曲线参数
   - 大数模运算（依赖bc，声称纯bash）
   - 椭圆曲线点加倍和加法
   - 标量乘法（二进制算法）
   - ECDSA签名生成

3. **用户界面**
   - ASCII艺术logo
   - 彩色警告信息
   - 进度条显示
   - 多种操作模式

### 故意保留的"缺陷"
1. **性能问题**
   - 单次签名380秒
   - 验证功能未实现
   - 标量乘法O(n)复杂度

2. **安全问题**
   - 熵质量不足（80位 vs 256位要求）
   - 时序攻击漏洞
   - k值重用风险
   - 无侧信道保护

3. **代码质量**
   - 依赖外部bc命令
   - 复杂的导入顺序
   - 讽刺性注释
   - 故意低效的算法

## 🚀 使用方法

### 基本操作
```bash
# 生成密钥对（约120秒）
./becc.sh genkey

# 签名文件（约380秒）
./becc.sh sign message.txt

# 性能对比测试
./becc.sh benchmark

# CPU加热模式
./becc.sh heat
```

### Docker运行
```bash
docker build -t beccsh .
docker run -it --rm beccsh genkey
```
